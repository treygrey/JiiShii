# JiiShii Game Package

This folder is a starter package for a drop-in JiiShii game. Copy it over
`src/game/` for bundled web development, or ship it as a sibling `game/` folder
beside a desktop player executable. Add scenes, assets, audio, sprites, and
optional surface modules without editing engine source.

## Required Files

- `game.config.js` owns the title, first scene, storage namespace, shell labels,
  and audio scene presets.
- `vn.js` is the local author import shim. Scene files import commands from it.
- `game.manifest.json` lists loose-package files for the desktop player.
- `scenes/` contains auto-discovered scene modules.
- `assets/` contains discovered images, audio, and sprite folders.
- `surface-modules/` contains optional scripted surfaces such as phone galleries,
  browsers, laptop screens, or other game-specific modules.
- `sprite-animations.js` registers optional game-specific IRL sprite transition
  presets before scene validation.
- `sprite-manifest.json` is generated by `npm run gen:sprites`.

When this package is used as a loose desktop `game/` folder, keep
`game.manifest.json` checked in. In dev mode the player can rebuild it when
files change; in release mode changed script/config files produce warnings.

Scene modules should import from the local shim:

```js
import { scene, stage, say } from "../vn.js";
```

Files ending in `.example.js`, `.test.js`, `.spec.js`, or starting with
`_private` are ignored by discovery.
